# 🤝 Contributing Guide

Thank you for your interest in contributing to this project !

We welcome contributions of all kinds, including bug reports, feature requests, documentation improvements, and code enhancements.

---

## 🚀 How to Contribute

1. Fork the repository
2. Clone repository
3. Create a new branch

   for example : `feature/<your_feature_name>`

4. Make your changes

   - Follow existing code style
   - Keep code clean and readable
   - Add comments if necessary

5. Commit your changes  
   Use clear and descriptive commit messages.

6. Push changes
7. Create a Pull Request  
   Submit changes for review and merging.

---
## 🐛 Reporting Issues

If you encounter a bug or issue:

- Use a clear title
- Describe the steps to reproduce the issue
- Include additional details if needed

---

## 💡 Feature Requests

New ideas are welcome!

Describe:
- The problem you want to solve
- Why this feature is useful
- Implementation overview (optional)

---

## 🧭 Development Guidelines

- Use modern C++ (C++20)
- Avoid memory leaks (use smart pointers)
- Keep the application lightweight
- Be consistent across files and code structures

---

## 🙌 Code of Conduct

Keep communication polite, open, and constructive.

---
Thank you for contributing 🚀
